# Contributing to BackupX

Thank you for your interest in contributing to BackupX!

## Getting Started

1. Fork the repository
2. Clone your fork: `git clone https://github.com/<your-username>/BackupX.git`
3. Create a feature branch: `git checkout -b pr/your-branch-name`

## Making Changes

1. Keep changes focused and atomic
2. Follow existing code style and conventions
3. Test your changes thoroughly
4. Commit with clear, descriptive messages

## Pull Request Process

1. Update documentation if needed
2. Ensure all tests pass
3. Open a pull request against the `main` branch
4. Reference any related issues in your PR description

## Code of Conduct

Please be respectful and constructive in all interactions.

## Questions?

Feel free to open an issue for any questions or discussions.
